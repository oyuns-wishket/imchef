# 기능 04 — 로고/브랜드 마크

> Imchef(레시피 공유 SNS)의 헤더 마크와 워드마크를 브랜드에 맞는 새 로고로 교체한다.
> 이 기능은 **브랜드 비주얼 자산 교체**이므로 데이터/API가 없다. 본 명세는 *요구사항·제약·시안 방향* 만 확정하며,
> 실제 5개 시안 렌더는 다운스트림(`frontend-design`)이 담당한다(00-feature-delta-plan §3.3).

---

## 1. 개요

- **해결하는 문제**: 현재 헤더와 파비콘에 쓰이는 `Ladle`(국자) 아이콘이 실제로는 **돋보기(virtual magnifier)와 동일한 실루엣**으로 그려져 있다. `icons.tsx`의 `Ladle`은 `circle(cx10,cy10,r6.5)` + 두 개의 사선 `line`으로 구성되는데, 이는 같은 파일의 `Search` 아이콘(`circle(cx11,cy11,r8)` + 사선)과 시각적으로 거의 구분되지 않는다. 더구나 `BottomNav`에 실제 검색용 돋보기(`Search`) 아이콘이 동시에 존재하므로, **헤더의 "국자=브랜드 마크"가 하단의 "돋보기=검색"과 혼동**된다. 이로 인해:
  - 사용자가 브랜드 마크를 검색 버튼으로 오인하거나, 앱의 정체성(요리/레시피)을 첫인상에서 인지하지 못한다.
  - 18~26px 소형 헤더 마크에서 "국자"라는 의도가 전혀 전달되지 않아 브랜드 회상(brand recall)이 약하다.
  - 파비콘/앱 아이콘(브라우저 탭, 홈 화면, OG 썸네일)에서도 동일 마크가 쓰여 정체성 약화가 전 채널로 전파된다.
- **제공하는 가치**:
  - **식별성**: 헤더 18px에서도 돋보기와 명확히 구분되는, 요리/셰프 의미가 읽히는 실루엣 → 첫인상 브랜드 인지율 개선.
  - **일관성**: 헤더 마크 ↔ 워드마크 ↔ 파비콘 ↔ OG 이미지가 단일 마크로 통일 → 채널 간 브랜드 연속성.
  - **접근성**: 흑백 라인 단색(currentColor)으로 라이트 배경에서 충분한 대비 확보, `aria-label` 유지.
  - **유지보수성**: 기존 `icons.tsx` 패턴(viewBox 0 0 24 24 / stroke currentColor)을 그대로 따라 사이즈·색상이 부모 컨텍스트에 자동 상속.
- **범위**:
  - **다루는 것**:
    - 헤더 브랜드 마크(아이콘) 교체 — `icons.tsx`의 `Ladle` 자리.
    - 워드마크 "Imchef" 조판(현 `Im` + `chef` 강조 유지/조정)과 마크의 조합 규칙(lockup).
    - 파비콘/앱 아이콘 자산 일관화(`favicon.ico` / `app/icon.svg` / `apple-icon`).
    - OG/공유 썸네일에 쓰이는 마크 표현(정적 자산 또는 OG 템플릿 내 마크).
    - 5개 시안 방향(모티프/스타일)의 요구사항·제약 정의(시안 렌더는 다운스트림).
  - **다루지 않는 것**:
    - 시각 디자인 최종 결정(픽셀/곡률/획 두께 미세값) — `frontend-design` 산출물 + 사용자 선택(빌드 단계).
    - 컬러 팔레트 확장 — 흑백 라인 미감 고정(00-feature-delta-plan §3.3, 새 컬러 액센트 도입 금지).
    - 검색 아이콘(`Search`)·하단 네비 모핑 동작(기능 01에서 다룸).
    - 로딩/스피너 마크 모션(기능 03 로딩 표준).
    - 마케팅 전체 BI 가이드(명함·간판 등 오프라인) — 본 기능은 앱 내 디지털 자산 한정.

## 2. 사용자 스토리 + 인수 조건

### US-1: 브랜드 마크 식별성

As a **Imchef 첫 방문 사용자**, I want **헤더의 브랜드 마크를 보고 "요리/레시피 앱"임을 즉시 인지**, so that **돋보기로 오인하지 않고 앱 정체성을 첫인상에서 파악한다**

**인수 조건:**
- [ ] AC-1: 새 마크는 헤더 렌더 크기(26px, `strokeWidth={1.5}`)와 소형 크기(18px)에서 모두 식별 가능해야 하며, **현재 `Ladle`/`Search`와 다른 실루엣**(원+사선 조합이 아닌 형태)으로 그려진다. (시각 회귀: 새 마크 SVG에 `circle` 단독 + 대각 `line` 두 줄 조합이 그대로 남아 있지 않다.)
- [ ] AC-2: 새 마크는 요리/셰프 모티프(냄비·프라이팬·셰프모자·뒤집개·숟가락/포크 등 §6 후보 중 택1) 중 하나의 실루엣이 18px에서 인지되도록, 내부 디테일 요소 수가 과밀하지 않다(권장 path/line 요소 5개 이하, 최소 획 간격 ≥ 1.5 user unit @ 24 viewBox).
- [ ] AC-3: `Ladle`을 import 하던 모든 사용처(`Header.tsx`)가 신규 마크 컴포넌트로 교체되어도 빌드 에러 없이 렌더되고, `aria-label="홈"`(헤더 Link) 접근성 레이블이 유지된다.

### US-2: 채널 간 일관성 (마크 ↔ 워드마크 ↔ 파비콘 ↔ OG)

As a **재방문/공유 경유 사용자**, I want **브라우저 탭·홈 화면 아이콘·공유 카드에서 헤더와 같은 마크를 본다**, so that **여러 접점에서 같은 브랜드로 인식한다**

**인수 조건:**
- [ ] AC-4: 헤더 마크와 파비콘(`app/icon.svg` 또는 `favicon`)이 **동일한 기본 형태**(같은 실루엣)를 사용한다. 파비콘은 16/32px에서 식별 가능하도록 단순화 버전(획 더 굵게/디테일 축소)을 허용하되, 헤더 마크와 같은 모티프를 유지한다.
- [ ] AC-5: 워드마크는 현행 조판(`Im` 일반 + `chef` 강조)을 기본으로 하며, 마크+워드마크 lockup의 가로 정렬(아이콘 좌, 텍스트 우)과 간격(현 `gap-2`)이 유지되거나 시안에서 정의한 값으로 잠긴다(lock). 마크 단독 사용 시(파비콘/앱 아이콘) 워드마크 없이도 성립한다.
- [ ] AC-6: OG/공유 썸네일에 마크가 포함될 경우, 동일 마크 형태가 흑백 라인으로 렌더되어 헤더와 시각 연속성을 가진다(별도 컬러/그라데이션 도입 금지).

### US-3: 흑백 라인 미감 + 접근성 준수

As a **디자인 시스템 유지보수자**, I want **새 마크가 기존 흑백 라인 토큰·아이콘 패턴을 그대로 따른다**, so that **사이즈/색상이 부모에 상속되고 디자인 일관성이 깨지지 않는다**

**인수 조건:**
- [ ] AC-7: 신규 마크 SVG는 `icons.tsx`의 `base` 규칙(`viewBox="0 0 24 24"`, `fill="none"`, `stroke="currentColor"`, `strokeLinecap/Linejoin="round"`)을 준수하고, 고정 색상(hex/inline)을 SVG 내부에 하드코딩하지 않는다 → 라이트 배경에서 `text-ink`(#141414) 라인으로 렌더된다.
- [ ] AC-8: 마크 컴포넌트는 `SVGProps<SVGSVGElement>`(현 `type P`)를 받아 `className`/`strokeWidth`/`aria-*`를 외부에서 주입할 수 있고, props로 사이즈 오버라이드(`w-[..] h-[..]`)가 적용된다.
- [ ] AC-9: 마크가 의미 전달 요소가 아닌 장식일 때(워드마크가 함께 있을 때)는 `aria-hidden` 적용이 가능해야 하며, 마크 단독(파비콘 외 인앱 단독 사용 시)일 때는 접근성 텍스트가 제공된다.

### US-4: 반응형/상태별 표현 일관성

As a **모바일/데스크탑 사용자**, I want **뷰포트와 사용 맥락(로딩·빈상태 등)에 따라 마크가 적절히 표현된다**, so that **어떤 화면에서도 깨지거나 흐릿하지 않다**

**인수 조건:**
- [ ] AC-10: 마크는 벡터(SVG)이므로 18~512px(앱 아이콘) 전 범위에서 래스터화 흐림 없이 선명하게 렌더된다.
- [ ] AC-11: 빈 상태(검색 결과 0건 등)·로딩 placeholder에서 브랜드 마크를 워터마크로 쓸 경우, `text-ink-faint`(#9a9a9a) 등 약한 톤으로 currentColor 상속만으로 표현 가능하다(마크 자체에 별도 변형 자산 불필요).

## 3. UI/UX 명세

### 컴포넌트 계층 (사용처 카탈로그)

```
브랜드 자산 (단일 마크 SSOT)
├── icons.tsx
│   └── <Logo /> (신규 마크 컴포넌트 — 구 Ladle 자리)
├── Header.tsx
│   └── Link[홈] (aria-label="홈")
│       ├── <Logo className="w-[26px] h-[26px]" strokeWidth={1.5} aria-hidden />
│       └── <span> 워드마크 "Im" + <span.accent>"chef"</span>
├── app/icon.svg / app/apple-icon (파비콘·앱 아이콘 — 마크 단독, 단순화 버전)
├── app/opengraph-image (OG 썸네일 — 마크 + 워드마크 lockup)
└── (선택) EmptyState / Loading placeholder
    └── <Logo />를 워터마크(faint 톤)로 재사용
```

### 5가지 화면 상태 (사용 맥락별 표현으로 매핑)

> 본 기능은 데이터 fetch가 없는 정적 자산이므로, 표준 5상태를 **마크가 등장하는 사용 맥락별 표현**으로 매핑한다.

| 상태 | 조건 (사용 맥락) | UI 표현 | 사용자 액션 |
|------|------------------|---------|------------|
| Empty | 검색/내 레시피 결과 0건 화면 | 마크를 `text-ink-faint` 워터마크로 중앙 배치 + 안내 문구 → 브랜드 잔상 유지 | 빈 상태 안내 따라 작성/검색 |
| Loading | 페이지 초기 진입(셸 렌더 중) | 헤더 마크는 정적으로 즉시 노출(스켈레톤화하지 않음) — 브랜드는 항상 선명 | 로딩 대기 |
| Partial | 파비콘/OG는 로드, 헤더 마크 폰트 늦게 로드 | 마크(SVG 인라인)는 즉시, 워드마크 텍스트는 폰트 폴백(Pretendard 미로드 시 system-ui)으로 우선 렌더 | 자동 보정(폰트 로드 후 swap) |
| Error | 마크 자산 로드 실패(외부 파비콘/OG fetch 실패) | §8 폴백: 인라인 SVG는 실패 불가, 파비콘 실패 시 브라우저 기본 → OG 실패 시 텍스트-only 카드 | 영향 없음(자동 폴백) |
| Ideal | 정상 — 헤더/파비콘/OG 전부 동일 마크 | 헤더: 마크+워드마크 lockup, 흑백 라인. 파비콘: 단순화 마크. OG: 마크+워드마크+타이틀 | 홈 이동(헤더 Link), 공유 |

### 반응형 대응

- **Mobile (< 768px)**: 헤더 마크 26px(현행) 유지, 워드마크 18px(`text-[18px]`). 앱 컨테이너 `max-w-[520px]` 모바일 우선 레이아웃이므로 마크는 좌측 고정, 터치 타깃은 Link 전체(높이 60px 영역).
- **Tablet (768px ~ 1024px)**: 동일 lockup 유지(컨테이너 중앙 정렬). 마크/워드마크 비율 변화 없음.
- **Desktop (> 1024px)**: 동일. 고DPI(레티나)에서 SVG 벡터로 자동 선명. OG 이미지는 1200×630 권장 규격으로 마크 확대 사용.

### 인터랙션 상세

| 인터랙션 | 동작 |
|----------|------|
| 헤더 로고 클릭/탭 | `/`(홈)로 이동(현 `Link href="/"`). active 시 미세 피드백은 현행 컨테이너 스타일 유지(별도 애니메이션 추가 없음). |
| 호버(데스크탑) | 마크 색상은 `text-ink` 상속 유지(돋보기 버튼처럼 hover 배경 토글 없음 — 브랜드 마크는 항상 ink). |
| 폰트 로드 swap | 워드마크 텍스트만 폰트 swap(마크는 SVG라 영향 없음) — 레이아웃 시프트 최소화. |

### 시각 디자인 스펙 (제약만 — 픽셀 결정은 다운스트림)

| 항목 | 값/제약 | 비고 |
|------|---------|------|
| 색상 | `currentColor` → 기본 `--color-ink` #141414 | 하드코딩 금지(AC-7) |
| 약한 톤(워터마크) | `--color-ink-faint` #9a9a9a | currentColor 상속만으로 |
| viewBox | `0 0 24 24` | icons.tsx 표준 |
| stroke-width | 헤더 1.5, 기본 1.7, 파비콘 단순화 시 ≥ 2.0 허용 | 소형 식별성 위해 파비콘은 더 굵게 |
| 코너 | `round` (linecap/linejoin) | icons.tsx 표준 |
| 최소 식별 크기 | 18px | AC-1 |

## 6. 비즈니스 로직

> 본 기능의 "비즈니스 로직"은 데이터 처리가 아니라 **브랜드 컨셉 방향**이다. 5개 시안의 모티프/스타일 후보와 네이밍("Imchef" = "I'm chef") 활용 아이디어를 정의한다.

### 브랜드 컨셉 방향 (모티프 후보)

"Imchef = I'm chef"는 "나는 셰프다 / 누구나 셰프"라는 자신감·일상요리의 메시지. 흑백 라인 단색에서 18px 식별이 가능해야 하므로, **단순·상징적 실루엣**을 우선한다.

| # | 모티프 방향 | 컨셉 설명 | 식별성 메모 |
|---|------------|----------|------------|
| A | 냄비/소스팬(pot/saucepan) | 손잡이 달린 냄비 실루엣 ± 김(steam) 라인. 요리의 보편 상징. | 손잡이+본체로 돋보기와 즉시 구분, 18px 안전. |
| B | 셰프 모자(toque) | 둥근 셰프 모자 라인. "I'm chef" 정체성 직결. | 상단 주름은 소형에서 단순화 필요(획 수 주의). |
| C | 뒤집개/주걱(spatula·turner) | 사선 도구 라인. 동적·요리 행위 강조. | 가늘면 검색 사선과 혼동 위험 → 헤드 형태 강조 필수. |
| D | 숟가락+포크(cutlery, 식기) | 식사/공유의 상징, SNS "함께 먹다" 뉘앙스. | 두 도구 병치는 소형에서 과밀 주의(획 ≤ 5). |
| E | 모노그램 "I"+셰프 요소 | 워드마크 첫 글자 "I"를 셰프모자/냄비와 결합한 레터마크. "I'm chef" 말장난 시각화. | 레터마크는 파비콘 단독성 우수, 워드마크와 중복 주의. |

> 위 5개는 §11에서 `frontend-design` 시안 5개로 1:1 매핑된다. 사용자가 1개 선택 → 토큰/마크업 lock.

### "I'm chef" 말장난 활용 아이디어 (워드마크 레이어)

- 현행 조판 `Im` + `chef`(강조) 유지를 기본으로, 시안에 따라 `I'm chef` 어포스트로피 노출/마크가 "I"를 대체하는 레터마크(방향 E) 변형 허용.
- 마크와 워드마크는 분리 가능(파비콘=마크 단독, OG=마크+워드). 어느 조합에서도 흑백 라인 미감 유지.

### 유효성 검사 (자산 산출 기준 — 디자인 게이트)

| 항목 | 규칙 | 실패 시 |
|------|------|--------|
| viewBox | `0 0 24 24` 고정 | 시안 반려(icons.tsx 비호환) |
| 색상 | SVG 내 hex/rgb 하드코딩 없음, `stroke="currentColor"` | 시안 반려(AC-7) |
| 식별성 | 18px 렌더에서 돋보기와 구분되는 실루엣 | 시안 반려(AC-1) |
| 획 밀도 | path/line 요소 ≤ 5, 최소 획 간격 ≥ 1.5 user unit | 단순화 요청 |
| 단독 성립 | 워드마크 없이 마크만으로 브랜드 식별(파비콘) | 레터마크/단순화 보강 |

### 엣지케이스

- **케이스 1 (소형에서 디테일 붕괴)**: 24 viewBox 디테일이 16/18px에서 뭉개짐 → 파비콘 전용 단순화 버전(획 ↑, 디테일 ↓)을 별도 산출(AC-4). 헤더용/파비콘용 2 변종 허용, 모티프는 동일.
- **케이스 2 (다크모드/반전 배경)**: 현재 라이트 단일 테마지만, 향후 반전 배경에서 마크가 안 보일 위험 → `currentColor` 상속만 쓰므로 부모 텍스트색 변경 시 자동 대응(별도 자산 불필요). 고정색 하드코딩이 이 안전성을 깨므로 금지(AC-7).
- **케이스 3 (워드마크 폰트 미로드)**: Pretendard CDN 미로드 시 워드마크가 system-ui로 폴백 → 마크(SVG)는 영향 없음, lockup 정렬만 미세 변동 → `gap`/baseline은 폰트 메트릭에 견고하도록 flex 중앙 정렬 유지.
- **케이스 4 (기존 `Ladle` import 잔존)**: 교체 후 `Ladle`을 참조하는 코드가 남으면 빌드 에러 또는 구 마크 노출 → 전역 grep으로 `Ladle` 사용처(현재 `Header.tsx`만) 전수 교체, 하위호환 필요 시 `export const Ladle = Logo` 별칭(deprecated)으로 회귀 방지(AC-3).

## 7. 상태 관리

**해당 없음 — 정적 마크.** 클라이언트/서버 상태 없음. React Query·SWR 미사용. 낙관적 업데이트 해당 없음(읽기·렌더 전용 비주얼 자산). 마크는 props로만 제어(`className`/`strokeWidth`/`aria-*`)되는 순수 프레젠테이션 컴포넌트다.

## 8. 에러 처리

> 인라인 SVG 마크는 번들에 포함되어 "로드 실패"가 발생하지 않는다. 에러 처리는 **외부 정적 자산(파비콘·OG·폰트)의 로드 실패 폴백**에 한정된다.

| 시나리오 | 에러코드 | 사용자 메시지 | 복구 전략 |
|----------|----------|--------------|----------|
| 인라인 마크(`<Logo/>`) 렌더 | — | (실패 불가) | SVG는 번들 동봉 → 네트워크 무관, 항상 렌더 |
| 파비콘 자산 404/로드 실패 | 404 | (사용자 무노출) | 브라우저 기본 파비콘으로 자동 폴백, 앱 동작 무영향 |
| OG 이미지 생성/로드 실패 | 500/404 | (공유처 무노출) | 메타 태그 폴백: 텍스트 title/description-only 공유 카드, OG 이미지 누락 허용 |
| 워드마크 폰트(Pretendard CDN) 미로드 | NETWORK_ERROR | (사용자 무노출) | `--font-sans` 폴백 체인(system-ui)으로 워드마크 렌더, 마크는 영향 없음 |
| 신규 마크 import 누락(빌드) | BUILD_ERROR | (사용자 무노출) | 빌드 타임 검출 → `Ladle` 별칭 export로 회귀 방지(케이스 4) |

## 11. 이미지 에셋 요구사항

> 본 기능의 핵심 산출물. `frontend-design`이 **5개 시안 방향(§6 A~E)** 을 흑백 라인 SVG로 렌더하고, 사용자가 1개를 선택해 교체한다(00-feature-delta-plan §3.3 시안 정책).

### 5개 시안 방향 (모티프 ↔ 스타일 매핑)

| 시안 # | 모티프(§6) | 스타일 방향 | 산출 형태 |
|--------|-----------|------------|----------|
| 시안 1 | A 냄비/소스팬 | 미니멀 라인, 김(steam) 1~2줄 옵션 | 헤더용 + 파비콘 단순화 |
| 시안 2 | B 셰프 모자(toque) | 둥근 라인, 주름 최소화 | 헤더용 + 파비콘 단순화 |
| 시안 3 | C 뒤집개/주걱 | 동적 사선 + 헤드 강조 | 헤더용 + 파비콘 단순화 |
| 시안 4 | D 숟가락+포크 | 대칭 식기 라인, 과밀 회피 | 헤더용 + 파비콘 단순화 |
| 시안 5 | E "I" 레터마크 + 셰프요소 | 레터마크/모노그램 | 헤더용 + 파비콘 단독성 강조 |

### 산출 포맷·제약

| 용도 | 설명 | 방법 | 포맷/제약 |
|------|------|------|----------|
| 헤더 마크 | 워드마크 옆 26px(@1.5 stroke) | `frontend-design` SVG → `icons.tsx` 컴포넌트 | viewBox 0 0 24 24, `stroke="currentColor"`, fill none, round cap/join, 색상 하드코딩 금지 |
| 파비콘 | 브라우저 탭 16/32px | 동일 모티프 단순화 SVG | `app/icon.svg`(권장) — 단색 currentColor, 16px 식별 위해 stroke ≥ 2 |
| 앱 아이콘(iOS/Android) | 홈 화면 | 동일 마크 PNG/SVG | `app/apple-icon`(180×180), 패딩 safe-area 고려 |
| OG/공유 썸네일 | 링크 공유 카드 | 마크+워드마크 lockup | `app/opengraph-image`(1200×630), 흑백 라인, 별도 컬러 금지 |
| (선택) 빈상태 워터마크 | EmptyState 배경 | `<Logo>` 재사용 | currentColor faint 톤, 신규 자산 불필요 |

### `frontend-design` 호출 계획 (다운스트림/빌드 단계)

1. 빌드 단계에서 `frontend-design:frontend-design` 호출 → 위 5개 시안(§6 A~E)을 **한 화면에 비교 렌더**(흑백 라인, 헤더 26px·파비콘 16px·OG 미리보기 3 크기로 동시 노출).
2. 제약 주입: globals.css 토큰(`--color-ink`/`--color-ink-faint`), `icons.tsx` `base` 규칙, AC-1/AC-2/AC-7 검증 기준.
3. 사용자가 **1개 선택** → 해당 시안의 SVG path를 `icons.tsx` 신규 컴포넌트로 lock.
4. 선택 마크의 파비콘 단순화 버전·OG lockup을 파생 산출.

### 교체 대상 파일 (touch set)

| 파일 | 변경 내용 |
|------|----------|
